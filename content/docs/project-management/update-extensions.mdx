# Update Codex and Extensions

Codex has two update layers:

1. The **Codex desktop app** updates the application itself.
2. The **Codex extension profile** controls the translation extensions used by a project.

Most users should update the app normally and let Codex manage project-specific extension versions automatically.

## Update the Codex App

Codex checks for app updates automatically. To check manually:

1. Open the **Main Menu**.
2. Go to **Tools**.
3. Click **Updates**.
4. Follow the prompt to install the update.
5. Restart Codex if prompted.

You can verify the app version from **Help > About Codex** on Windows/Linux or **Codex > About Codex** on macOS.

## Understand Extension Pins

Some projects require specific versions of the Codex Translation Editor, Frontier Authentication, and related extensions. Codex uses **CodexConductor** to manage these version pins.

When a project has pins:

- Codex may create or switch to a project-specific profile.
- Extensions in that profile may be installed from pinned VSIX releases.
- Codex may reload before the project opens.
- The translation editor may wait with a message like "Applying extension version pins" until the required versions match.

<Callout type="warning">
Do not manually update the Codex Translation Editor from the Extensions panel for a pinned team project unless your coordinator or support asks you to. The project may switch back to the pinned version.
</Callout>

## Manage Pins as a Coordinator

Project coordinators may use **Manage Extension Pins** from the command palette when the team needs a controlled extension version. Pins can also be managed by the Codex CLI for advanced workflows.

After changing pins, sync the project so teammates receive the update. Teammates need network access the first time Codex downloads a newly pinned extension version.

## Manual Extension Updates

Manual extension updates are mainly for unpinned projects or support-directed troubleshooting:

1. Open the Extensions panel.
2. Find **Codex Translation Editor**.
3. Check the installed version.
4. Update or install a specific version only if the project is not pinned or support has instructed you to do so.
5. Reload Codex.

## Troubleshooting Update Issues

### Codex is stuck applying extension pins

Stay online and wait for the profile switch or extension download. If Codex shows an error report, copy it and include it with your support request.

### Extension update does not stick

The project is probably pinned. Ask the coordinator to update the project pin instead of updating the extension manually.

### App update will not install

Restart Codex and try again. On Windows, make sure all Codex windows are closed before installing. If antivirus blocks the updater, see [Antivirus Blockages](/docs/antivirus-blockages).
