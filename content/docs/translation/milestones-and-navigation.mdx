# Milestones and Navigation

Milestones divide structured projects into sections that Codex can navigate, review, comment on, and export. In scripture projects, milestones often map to chapters or verse ranges. In media or document projects, they may represent sections, cues, or imported structure.

## Why Milestones Matter

Milestones help Codex:

- Show chapter or section navigation.
- Keep comments sorted by location.
- Group progress by structured ranges.
- Support source editing and cell movement workflows.
- Select ranges during some export workflows, especially Bible audio export.

## Navigate Structured Projects

Use the chapter or section controls at the top of the translation editor to move through milestones. The Navigation panel also shows imported files and progress so you can jump between sources and target files.

## Edit Milestone Labels

Some projects allow milestone names or labels to be edited inline. Use this carefully in shared projects because milestone changes can affect navigation, comments, export selection, and teammate expectations.

## Milestones and Export

Some export workflows add a **Select Milestones** step. This lets you export only selected chapters or ranges instead of the entire file group.

If you do not see milestone selection, the selected file group or export format may not need it.

## Best Practices

- Keep milestone labels stable after collaborators begin work.
- Use comments for questions tied to a specific location.
- Confirm milestone selection before exporting audio or scripture deliverables.
- Sync after structural changes in shared projects.
